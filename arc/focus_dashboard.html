cycle mapi
시스템의 모든 원자적 변화 1개 = 1스텝. ▶ = 현재 스텝. 클릭하거나 ← → (또는 ↑ ↓)로 이동.
박스 친 줄 = 큰 단계(input → propose → decide → apply → output)가 풀이의 구조. 그 아래 줄들이 그 단계 안에서 일어난 변화.
wave N(평문) = 그 단계 안의 elaboration sub-cycle. wave 1 = 정식 결정에 의한 1차 발화(propose의 operator 제안 / apply의 apply 규칙·body), wave 2·3… = 그 결과로 연쇄된 2차 wave. 같은 wave 번호 = 한 settle 라운드.
각 wave는 match → fire → wm-update 3원자단계로, cycle map에서 왼쪽 색 띠로 구분: match=주황(LHS 충족/깨짐 검출), fire/retract=보라(발화·철회를 한 색으로; rules 패널에선 초록/빨강 구분), wm-update=녹색(preference가 WME로 들어가 WM 변화). operator body(직접 적용)는 wave가 아니라 APPLY phase의 효과라 색 띠·wave 번호가 없다. WM은 wm-update에서만 바뀐다(match·fire 땐 불변). 발화했는데 뒤에 wm-update가 없으면 = 그 결과가 o-support라 제거할 게 없는 무해한 철회.
각 줄은 [wave][stage][detail] 고정폭 컬럼으로 정렬되고, detail이 길면 …로 잘린다 — 줄을 클릭하면 하단 바에서 전체 텍스트를 본다.

working memoryi
S1을 루트로 한 단일 WM 트리. 모든 줄 = (id ^attr value) WME 삼중쌍. ▸ 토글로 lazy 하위(pair·grid·object) 펼침. 우측 ⇕ all = 전부 열기(일부라도 닫혀 있으면)/전부 닫기. 한 토글의 열림 상태는 다음 step으로 유지되고, 다음 step에서 새로 생긴 토글은 닫힌 채로 시작.
색: 녹색 바탕=그 줄 추가, 빨강 바탕=그 줄 삭제. 하위에서만 바뀐 상위 토글은 글자색만 — 녹=추가/적=삭제/주황=혼합.
한 객체가 두 엣지로 가리켜질 때(예: 선택된 (S1 ^operator O1)와 그 acceptable preference (S1 ^operator O1 +)가 둘 다 O1을 가리킴) 하위는 먼저 나온 엣지 아래에 한 번만 펼치고, 나머지 엣지는 평범한 leaf로 둔다(중복 방지).

rulesi
에이전트의 기본 production 규칙 (절차적 지식). 이 step에 발화한 규칙 = 초록 테두리 ● fired, 철회된 규칙 = 빨강 테두리 ● retracted. match step에선 발화/철회 직전 상태로 ◌ matched(주황 점선=LHS 충족, 발화 대기)·◌ unmatched(붉은 점선=LHS 깨짐, 철회 대기) 표시. 철회는 LHS가 더는 만족되지 않아 instantiation이 사라진 것이라, 그 규칙의 조건이 회색(미충족)으로 보이는 게 정상이다. 조건은 종류별 배경색: 긍정(파랑 기) = WM에 있어야 함, 부정(빨강 기, 앞에 −) = WM에 없어야 함. 글자색: 회색=미충족, 충족되면 긍정 파랑/부정 빨강 진한색. THEN의 operator = 금색. 한 규칙은 모든 조건이 충족(=진한색)일 때만 발화. 카드는 기본 접힘(WM과 동일) — 클릭해 펼치면 다음 step으로 유지. 우측 ⇕ all로 전부 열기/닫기.

problemi
train 예제쌍(input → output)과 test 입력. 풀이의 입력 데이터(ARC task).

answer candidatesi
compose/output 단계에서 생성되는 답 후보들. 랭킹 제출 순서대로 내고, 틀리면 다음 후보(최대 3회). ✓ = 정답.
