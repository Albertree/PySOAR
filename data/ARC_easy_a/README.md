# easy000 시리즈 — Slice 커리큘럼

1×1 객체 도메인. 입력 = 한 색 셀, 출력 = 한 색 셀. 난이도 그래디드.

| 문제 | 메커니즘 | 모든 출력 동일 | Slice | G0(입력) 기여 |
|---|---|---|---|---|
| **a** | 완전 고정 출력 `(5,5)=2` (입력 무관) | ✅ | **1** | ✗ |
| **b** | 완전 고정 출력 `(0,0)=3` — *a 의 값-agnostic 쌍둥이* | ✅ | **1** | ✗ |
| **c** | 위치 고정 `(5,5)`, **색 = 입력색** | ✗ | 2 | 색 |
| **d** | 위치 고정 `(1,2)`, 색 = 입력색 | ✗ | 2 | 색 |
| **e** | 위치 = 우하단 모서리(격자 크기에서), 색 = 입력색 | ✗ | 2~3 | 색 + 위치 |
| **f** | 위치 고정 `(4,4)`, 출력크기 = 입력크기, 색 = 입력색 | ✗ | 2~3 | 색 + 크기 |
| **g** | 출력크기 고정 `5×5`(≠입력), 위치 `(0,0)`, 색 = 입력색 | ✗ | 2~3 | 색 + 크기 |

- **Slice 1 = a + b**: 둘 다 *완전 고정 출력* (모든 example G1 COMM → 공통 grid 복사).
  b 가 *다른 고정값* 이라 `PredictByAllPairCommOp` 의 값-agnostic(하드코딩 없음)을 검증.
- **c~g**: 출력이 입력에 의존(색·위치·크기) → G0 분석 필요 = Slice 2+.
- 주의: 문서(SLICE_1_DEV / wiki)의 "Slice 2 = easy000b" 는 이 재명명 후 **easy000c**
  (위치 고정 + 입력색 복사)에 해당한다.
