# flip 폴더 — Concept 설계

`flip` 폴더의 concept은 그리드 안 객체를 **대칭(반전)** 시키는 변환을 다룬다. 파일 포맷/네이밍/색상 규칙은 루트 `HARNESS.md`를 따른다.

> 아래 설계는 대화 중 발견된 몇 가지 논리적 모순을 사용자와 확인 후 정리한 결과다. 원 아이디어와 달라진 부분은 각주로 표시.

## 0. 전제 — 객체 크기

**객체 크기는 항상 1×1 초과(다중 셀)로 제한한다.** 1×1 객체는 어떤 축으로 반전해도 형태가 바뀌지 않아(대칭이 자명함) flip 고유의 의미가 없고, 위치만 바뀌는 경우는 `move` concept과 구분되지 않기 때문. *(원래 "1x1 이상"이었으나 1x1 제외로 확정)*

## 1. 기본 분류 (객체 개수)

| 분류 | 설명 |
|---|---|
| **F-단일** | 그리드에 반전 대상 객체가 하나뿐 |
| **F-다** | 그리드에 여러 객체가 존재, 그중 하나만 반전 대상 |

### 1-1. F-단일: 객체 색상 구성 (3종)

단일객체 케이스는 "어느 객체를 고를지"는 문제되지 않으므로, 대신 객체 내부의 **색상 구성**으로 하위 개념을 나눈다.

1. **단색** — 객체를 이루는 모든 셀이 같은 색
2. **부분 다색** — 색상 종류가 1보다 많고 객체 크기(셀 수)보다는 적음 (즉 중복되는 색이 있는 다색 객체)
3. **전체 다색** — 색상 종류 수 = 객체 크기(셀 수) (모든 셀이 서로 다른 색)

*(원문의 "1 / 2 / 2" 번호는 "1 / 2 / 3" 오타로 간주하고 정리함)*

### 1-2. F-다: 객체 선택 근거 (3종)

다객체 케이스는 `move`의 M4와 동일하게 아래 중 하나로 반전 대상을 특정한다.

1. **색상** — 특정 색으로 이루어진 객체만 대상
2. **크기** — 특정 크기 조건(예: 가장 큰 객체)을 만족하는 객체만 대상
3. **모양** — 특정 모양(shape) 조건을 만족하는 객체만 대상

> F-다의 각 객체는 기본적으로 **단색(monochrome)** 이라고 가정한다. "색상"을 선택 근거로 쓰려면 객체마다 대표색이 하나로 정해져 있어야 하기 때문 — 1-1의 "색상 구성" 하위분류는 F-단일에만 적용되고 F-다에는 적용하지 않는다.

## 2. 대상 객체 가로/세로 길이 (홀수/짝수)

flip 방향(§3)에 따라 **관련 있는 쪽의 길이만** 홀/짝을 판단한다.

- **세로축 대칭**(좌우 반전) → 가로(width) 길이의 홀/짝이 관련
- **가로축 대칭**(상하 반전) → 세로(height) 길이의 홀/짝이 관련
- **대각선 대칭**(우상향/우하향) → bbox가 정방형인 경우 가로=세로이므로 값이 하나로 수렴; 정방형이 아닌 경우(§3-1 참고, 이동 있음 케이스에서 가능) 가로·세로 둘 다 관련되므로 각각 독립적으로 홀/짝을 지정할 수 있다.

홀/짝 여부는 "축이 객체 정중앙의 셀을 지나는가(홀수) vs 두 셀 사이를 지나는가(짝수)"를 결정한다.

## 3. flip 방향 (4종)

1. **세로축 기준 대칭** (좌우 반전)
2. **가로축 기준 대칭** (상하 반전)
3. **우상향 대각선 기준 대칭**
4. **우하향 대각선 기준 대칭**

### 3-1. 정방형(bbox) 제약 — 이동 유무에 따라 다름

대각선 대칭(3, 4)의 bbox 정방형 요구는 **이동 유무(§4)에 따라 달라진다** (사용자 확인 완료):

- **이동 없음**일 때: 대각선 대칭은 객체 bbox가 **정방형이어야 함** (필수)
- **이동 있음**일 때: 대각선 대칭이어도 객체 bbox가 정방형이든 아니든 **상관없음** (제약 없음)

> 이 규칙 덕분에 §4의 "기준선 방향=대각선(bbox 비정방형 필수)"과 모순 없이 공존한다 — flip방향의 정방형 요구가 "이동 있음" 구간에서는 적용되지 않으므로, "이동 있음 + flip방향 대각선 + 기준선방향 대각선"에서는 기준선 쪽의 "비정방형 필수" 요구만 살아남는다.

## 4. 객체 반전 후 이동 유무

1. **이동 없음** — 객체는 반전(내부 픽셀 재배열)만 되고 bbox 위치는 원래와 동일
2. **이동 있음** — 반전과 별개로 bbox 위치 자체가 달라짐 (아래 §4-1 "기준선"을 기준으로 위치가 재계산됨)

### 4-1. 이동 있을 시: 기준선(reflection line) 규칙

"기준선"은 이동 있음일 때만 존재하는, 객체의 **위치**를 반사시키는 데 쓰이는 가상의 선이다 (§3의 flip 방향이 객체 **내부 픽셀 패턴**을 반전시키는 것과는 별개의 축).

**기준선 위치**
1. **bbox 내부** — 기준선이 객체 bbox와 겹침
2. **bbox 외부** — 기준선이 객체 bbox와 겹치지 않음

**기준선 방향**
1. 가로
2. 세로
3. 우상향 대각 — 객체 bbox **비정방형 필수**
4. 우하향 대각 — 객체 bbox **비정방형 필수**

> §3-1에 따라 이 "비정방형 필수" 제약은 이동 있음 케이스에서만 존재하는 축이므로 flip방향의 정방형 제약과 충돌하지 않는다.

## 5. 조합 규모 — 전체 생성 완료

전체 조합 수 = 객체분류(F-단일 3종 + F-다 3종 = 6) × 홀짝(2) × flip방향(4) × 이동유무(없음1 + 있음: 기준선위치2×기준선방향4=8, 합9) = **6×2×4×9 = 432 조합 = 432커플 = 864 JSON 파일**. 사용자 확인 후 전량 생성 완료 (§7 참고).

## 6. 파일 네이밍

HARNESS.md §9 규칙을 그대로 따른다: `flip000a`(2ex) / `flip000b`(3ex) 부터 시작해 알파벳이 `z`를 넘어가면 `aa, ab, ...`로 확장(432커플 = 864 유닛이라 3자리 접미사까지 사용됨, 예: `flip000aaa`).

## 7. 생성 결과 및 구현 노트

432개 조합 전체를 스크립트로 생성했다. 전체 매핑(파일명 ↔ 조합 파라미터)은 `flip/MANIFEST.csv`에 있다 (idx, file_2ex, file_3ex, object_branch, sub, parity, flip_dir, move, line_pos, line_dir 컬럼).

구현 시 확정한 세부 규칙:

- **도형 생성**: 객체는 지정된 bbox(h×w)를 정확히 채우는 4-연결(connected) 폴리오미노를 랜덤 생성. 대각선 flip/기준선 제약(§3-1, §4-1)에 따라 h,w를 정방형/비정방형·홀짝 조건에 맞게 고른 뒤 도형을 채운다.
- **기준선(이동) 계산**: flip방향(§3)은 객체 내부 픽셀 패턴만 반전시키고, 기준선(§4-1)은 원본 bbox의 좌상단 앵커 좌표를 선/대각선에 대해 점 반사(reflection)시켜 반전된 패턴을 배치할 새 위치를 정한다. 두 변환은 독립적으로 합성된다.
- **다객체 선택근거 구현**: 색상=파일 전체에서 타깃 색 고정, 크기=타깃이 항상 distractor보다 넓은 면적, 모양=파일 내 타깃 도형(cell 패턴)을 한 번 고정해 모든 예시에 재사용 — 나머지 속성(색상 등)은 예시마다 랜덤화해 해당 속성이 선택 기준이 아님을 보장.
- **couple 색상 차이(HARNESS §7)**: `move`와 동일하게 2ex/3ex 파일마다 겹치지만 다른 5색 팔레트를 배정하고, 각 파일의 첫 예시 첫 색상을 팔레트의 고정 원소로 강제해 색상 집합이 항상 달라지도록 보장.
- 노이즈 방지: 예시가 실제로 아무 변화도 없는(no-op) 경우는 재시도하도록 처리했다.

**검증**: 864개 파일 전수 자동 검증(그리드 4~10, 배경0, 색 0-9, pair 내부 크기 고정, test 1개, 2ex=train2/3ex=train3, couple 색상 차이, 예시 no-op 없음, 컬러 픽셀 수 입출력 동일) 통과. 대표 샘플(단일/mono/vertical, 단일/mono/diag_dl no-move, 단일/mono/diag_ur+대각기준선 이동, 다객체/shape, 다객체/size, 다객체/color) 6종을 직접 grid로 출력해 flip·이동 로직이 의도대로 동작함을 확인.
