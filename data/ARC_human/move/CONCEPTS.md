# move 폴더 — Concept 설계

`move` 폴더의 concept은 **객체 개수 × 객체 크기** 2축으로 4종 분류한다. 파일 포맷/네이밍/색상 규칙은 루트 `HARNESS.md`를 따른다.

## 1. 기본 분류 (4 concept)

| ID | 객체 개수 | 객체 크기 | 설명 |
|---|---|---|---|
| **M1** | 단일객체 | 1×1 | 그리드에 색칠된 셀이 하나뿐인 경우 |
| **M2** | 단일객체 | 1×1 초과 | 그리드에 여러 셀로 이루어진 객체가 하나뿐인 경우 |
| **M3** | 다객체 | 1×1 | 1×1 크기 객체(셀)가 여러 개 존재 |
| **M4** | 다객체 | 1×1 초과 | 여러 셀로 이루어진 객체가 여러 개 존재 |

M1/M3 = 객체 크기가 항상 1×1. M2/M4 = 객체 크기가 1×1보다 큼(다중 셀 객체).

## 2. 객체 선택 근거 (다객체 문제에서 "어느 객체를 움직일지" 판단 기준)

다객체 문제(M3, M4)는 그리드 안에 이동 대상 후보가 여럿이므로, 어떤 객체가 이동 대상인지 규칙이 필요하다.

- **M3 (다객체, 1×1)**
  1. 고정 색 — 특정 색의 셀만 이동 대상
- **M4 (다객체, 1×1 초과)**
  1. 고정 색 — 특정 색으로 이루어진 객체만 이동 대상
  2. 크기 — 셀 개수가 **고정된 상수 X인 객체**만 이동 대상. "가장 큰/작은 객체"처럼 상대적으로 정하지 않는다 — 이동하는 객체의 크기(셀 수)는 **같은 task(파일) 안의 모든 pair에서 항상 동일한 값 X**여야 한다(고정 색이 파일 전체에서 하나의 색으로 고정되는 것과 같은 방식). distractor는 크기가 X가 아닌 객체.
  3. 모양 — 특정 모양(shape) 조건을 만족하는 객체

## 3. 이동 목적지점 규칙 (1×1 크기 객체 문제: M1, M3 대상)

M1, M3처럼 이동 대상 객체가 1×1인 경우, "어디로 이동하는가"는 아래 3가지 중 하나로 정의한다.

1. **상대 위치 고정** — 선택된 객체의 원래 위치 기준 고정 오프셋으로 이동 (예: `(+1, -2)`처럼 모든 pair에서 동일한 델타 적용)
2. **절대 위치 고정** — 그리드 내 특정 좌표로 이동 (예: 모든 pair에서 목적지가 `(4, 2)`로 동일)
3. **그리드 우하단 고정** — 그리드 크기와 무관하게 항상 우하단 모서리로 이동

> 주의: pair마다 그리드 크기가 달라질 수 있으므로(HARNESS.md §3), 규칙 2/3의 "고정 좌표"는 그리드 크기에 따라 좌표값 자체가 달라질 수 있다(우하단 좌표는 grid마다 다름). 규칙 3은 이 상대성을 명시적으로 다루는 케이스.

> ⚠️ **예외**: 목적지 규칙이 **"절대 위치 고정"**인 문제는, 목적지 좌표의 의미가 흔들리지 않도록 **같은 task(파일) 안의 모든 grid(train + test, input + output) 크기를 동일하게 고정**한다 (즉 이 concept에 한해 HARNESS.md §3의 "pair마다 크기가 달라도 됨"을 적용하지 않음). §5의 "절대 위치 고정"도 동일.

## 4. 이동 기준점 규칙 (1×1 초과 크기 객체 문제: M2, M4 대상)

객체 크기가 1×1보다 큰 경우 "객체의 위치"를 하나의 점으로 정의할 필요가 있다. 객체의 bounding box(bbox) 기준으로 아래 중 하나를 **기준점(anchor)**으로 삼는다.

1. **bbox 좌상단** (top-left)
2. **bbox 우하단** (bottom-right)

## 5. 이동 목적지점 규칙 (1×1 초과 크기 객체 문제: M2, M4 대상)

§4에서 정한 기준점을 기준으로, 목적지는 아래 3가지 중 하나로 정의한다.

1. **상대 위치 고정** — 기준점 원래 위치 대비 고정 오프셋으로 이동 (예: 모든 pair에서 동일한 `(+dx, +dy)` 적용)
2. **절대 위치 고정** — 기준점이 그리드 내 특정 좌표로 이동 (모든 pair 공통 좌표)
3. **그리드 코너 고정** — 기준점 종류에 대응하는 그리드 코너로 이동
   - 기준점이 **좌상단**이면 → 객체를 그리드 **좌상단**으로 이동
   - 기준점이 **우하단**이면 → 객체를 그리드 **우하단**으로 이동

> ⚠️ 주의: 객체가 이동한 뒤 그리드 밖으로 벗어나지 않아야 한다. 객체 크기(bbox 크기)와 그리드 크기, 오프셋/좌표 값을 함께 고려해서 이동 후 전체 객체가 그리드 내부에 들어오도록 설계할 것 — 특히 규칙 1(상대 오프셋)과 규칙 2(절대 좌표)에서 pair별 그리드 크기가 다를 수 있음(HARNESS.md §3)을 감안.

## 5-1. 다객체 문제(M3, M4)의 객체 개수 및 색상 규칙

- **객체 개수는 고정되지 않는다.** "이동 대상 1개 + distractor 1개 = 총 2개"로 항상 고정되는 것이 아니라, distractor 개수가 pair마다 달라질 수 있다 (구현: distractor 1~2개, 즉 총 객체 수 2~3개 사이에서 무작위).
- **한 grid 안에서 모든 객체의 색은 서로 달라야 한다** (색 중복 금지). 즉 같은 grid 내에 같은 색을 가진 서로 다른 객체(연결 요소)가 2개 이상 존재해서는 안 된다 — 선택 근거가 색상이 아닌 경우(크기/모양)에도 동일하게 적용.

## 6. 미정 사항 (추후 지시 예정)

- 각 concept(M1~M4) × 선택근거 × 기준점 × 목적지 조합 중 실제로 몇 개를 파일로 만들지, 어떤 조합을 우선할지는 사용자가 이후 지정.
