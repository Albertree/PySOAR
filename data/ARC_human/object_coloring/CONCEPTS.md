# object_coloring 폴더 — Concept 설계

`object_coloring`(파일 코드: `objc_`) 폴더의 concept은 그리드 안 객체의 **위치·모양·크기를 전혀 바꾸지 않고, 선택된 객체의 색만 바꾸는** 변환을 다룬다. 파일 포맷/네이밍/색상 규칙은 루트 `HARNESS.md`를 따르고, 객체 선택 근거 축은 `move`/`flip`/`rotate`와 동일한 설계를 재사용한다.

## 배경 — 왜 이 카테고리가 필요한가

현재 solver DSL의 `coloring(selection, color)`은 input/output 픽셀 비교로 "이 위치는 색이 바뀌었다"가 이미 드러나야만 활성화된다. 즉 **객체 단위 속성(색/크기/모양/위치)만으로 "어떤 객체를 칠할지" 고르는 `selection` 규칙이 없어서**, "두 객체가 위치·모양·크기는 같고 색만 다르다"처럼 객체 매핑이 필요한 문제를 DSL이 아예 표현하지 못한다. 이 카테고리는 그 gap을 겨냥한 학습/평가용 문제를 만든다.

## 0. 전제

- 객체 크기는 **최소 2칸 ~ 최대 9칸**으로 제한한다 (단일객체 포함 모든 문제 공통).
- **input과 output은 색이 바뀐 객체를 제외하면 완전히 동일**하다 (위치·모양·크기 변화 없음).

## 1. 기본 분류 (객체 개수)

| 분류 | 설명 |
|---|---|
| **OC-단일** | 그리드에 객체가 하나뿐 |
| **OC-다** | 그리드에 여러 객체가 존재, 그중 하나 이상의 색이 바뀜 |

### 1-1. OC-단일

선택 근거가 필요 없다 (객체가 하나뿐이므로). 그 객체를 task 전체에서 고정된 상수 색 C로 칠한다.

### 1-2. OC-다: 객체 선택 근거 (4종)

1. **색** — 특정 색(고정 상수 T)을 가진 객체(들)가 대상
2. **크기** — 셀 개수가 고정된 상수 X인 객체(들)가 대상 (`move` §2와 동일하게 "가장 큰/작은"이 아니라 절대 상수)
3. **모양** — 특정 고정 모양(shape)을 가진 객체(들)가 대상
4. **위치** — 객체가 모두 **같은 방향의 1차원 선**(세로 Nx1 또는 가로 1xN, 한 task 안에서는 방향 통일)이고, **고정된 절대 좌표(행 또는 열 인덱스)**에 있는 객체(들)가 대상. "가장 왼쪽/오른쪽" 같은 상대적 순서가 아니라 절대 인덱스로 정의한다 (`move`의 "크기=상수" 재정의와 같은 맥락).

> ⚠️ **위치 선택근거의 그리드 크기 예외**: 절대 좌표(행/열 인덱스)의 의미가 흔들리지 않도록, 선택근거가 "위치"인 문제는 **task 내 모든 grid 크기를 동일하게 고정**한다 (`move` §3의 "절대 위치 고정" 예외와 같은 이유로 HARNESS.md §3을 이 concept에 한해 적용하지 않음).

### 1-3. OC-다: 색이 변화하는 객체 개수

**항상 단일** — 선택근거를 만족하는 객체는 정확히 하나이고, 그 객체만 색이 바뀐다. 다른 객체는 모두 선택근거 값이 달라야 한다 (모호성 방지). 한 grid 안 모든 객체는 서로 다른 색을 가진다 (예외 없음).

> *(원래 "복수" — 선택근거를 만족하는 객체가 2개 이상이라 여러 개가 동시에 칠해지는 버전 — 도 설계했었으나, 한 grid에서 복수 개체가 동시에 변화하는 카테고리는 전부 제거하기로 확정하여 "단일"만 남았다.)*

## 2. 새 색(paint color) 결정 방식

선택된 객체(들)를 칠하는 새 색 C는 **task 전체에서 고정된 상수**다 (`move`의 "고정 색" 선택근거와 같은 방식 — 매 예시 무작위로 정하지 않고 파일 시작 시 한 번 정해서 재사용). 선택근거=색인 경우 식별색 T와 새 색 C는 항상 다르다 (T ≠ C).

## 3. 조합 규모

OC-단일(1) + OC-다(선택근거 4종, 변화개수는 항상 단일) = **5 조합 = 5커플 = 10 JSON 파일**.

## 4. 파일 네이밍

HARNESS.md §9 규칙을 그대로 따르되, code 자체가 `objc_`(밑줄 포함)이다: `objc_000a`(2ex) / `objc_000b`(3ex) 부터 시작해 필요시 `aa, ab, ...`로 확장.
